◆ iftLinkFileSetsByFilename()

iftDict* iftLinkFileSetsByFilename ( const iftFileSet *  file_set1,
const iftFileSet *  file_set2 
)

Links two file sets by indexing them into a dict by their filenames.

It tries to index all pathnames from file set1 with file set2.

Ex:
file_set1 = ["./orig/0001.scn", "./orig/0002.scn", "./orig/0003.scn"]
file_set2 = ["./labels/0001.scn", "./labels/0002.scn", "./labels/0003.scn"]
out_dict = {"./orig/0001.scn": "./labels/0001.scn", "./orig/0002.scn": "./labels/0002.scn", "./orig/0003.scn": "./labels/0003.scn"}

Parameters
file_set1First file set.
file_set2Second file set.
Returns
Output dict.

◆ iftValidDataSetFilenameConvention()

bool iftValidDataSetFilenameConvention ( const char *  path)

Checks if the path's filename conforms to IFT's default nomenclature.

Author
Thiago Vallin Spina
Date
Mar 09, 2016

The IFT default nomenclature is <label>_<sample>_<suffix>.<ext>

Ex1.: 000002_000015_LeftLung.pgm

The suffix may also be empty to conform with the old convention, or be separated by multiple '_' characters

Ex2.: 000002_000015.pgm Ex3.: 000002_000015_Right_Lung.pgm

Parameters
pathThe path name.
Note
The old convention 000002_0000015.pgm is also respected.
